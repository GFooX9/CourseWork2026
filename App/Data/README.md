# 📁 Слой данных (Data Layer)

## 📌 Назначение
Обеспечивает персистентность (постоянное хранение) результатов работы пользователя, структурирование исторических логов исследований и долгосрочную фиксацию связей между объектами.

## ⚙️ Состав и зоны ответственности
* **`Basedata/`** — Изолированная директория реляционной базы данных **SQLite** (`vision_pro.db`).
* **`database_manager.py`** — Слой абстракции доступа к данным (DAO). Берет на себя сырые SQL-запросы (`INSERT`, `SELECT`, `DELETE`), фильтрацию истории по временным периодам (сегодня/неделя) и текстовый поиск по номерам протоколов.
* **`history.json`** — Резервное хранилище нереляционных данных для локального кэширования сессий.

## 🔒 Правила рефакторинга
Прямое обращение к SQLite из файлов интерфейса (UI) **запрещено**. Вся работа с базой данных ведется строго через методы класса `DatabaseManager` для предотвращения блокировок потоков БД.
