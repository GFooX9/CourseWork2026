# 📁 Вычислительное ядро ИИ (Business Logic Layer)

## 📌 Назначение
Сердце программного комплекса. Содержит всю математическую и нейросетевую логику приложения. Напрямую управляет инференсом моделей компьютерного зрения и OCR.

## ⚙️ Состав и зоны ответственности
* **`vision_engine.py`** — Главный оркестратор ядра. Координирует сквозной анализ кадра, связывая вызовы детектора и распознавателя текста.
* **`object_detector.py`** — Управление инференсом **YOLO11**. Отвечает за извлечение классов и уверенности из PyTorch-тензоров. Содержит менеджер горячей смены весов с принудительной очисткой памяти GPU (`torch.cuda.empty_cache()` и `gc.collect()`).
* **`text_recognizer.py`** — Подмодуль экстракции текста **EasyOCR**. Переведен в режим максимального сбора кириллических строк с безопасным парсингом полигонов координат.
* **`model_installer.py`** — Асинхронный менеджер скачивания (`QThread`). Взаимодействует с публичным репозиторием `Ultralytics/YOLO11` на **Hugging Face**, организует скачивание бинарных весов и их удаление.
* **`analytics_engine.py`** — Вычислительный модуль агрегации статистических данных для построения графиков.
* **`image_processor.py`** — Модуль низкоуровневой работы с матрицами изображений OpenCV (нарезка кропов, кэширование оригиналов).

## 🔒 Правила рефакторинга
Компоненты ядра должны быть **100% изолированы от GUI**. Импорт модулей `PyQt6` (кроме сигналов/потоков) в этот раздел запрещен. Ядро принимает на вход пути/матрицы, а возвращает чистые структуры данных Python (словари/списки).
